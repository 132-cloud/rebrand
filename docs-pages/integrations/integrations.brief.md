# Creative Brief — Integrations

**Page:** /integrations
**In one line:** A searchable, filterable directory of 73+ partner integrations that extend the Nymbus platform across key banking functions.

## What this page is for
Mid-funnel consideration and technical due diligence. Prospects evaluating Nymbus want to know it connects to their existing tools — card processors, fraud vendors, KYC providers, loan origination systems, and more. Primary action: validate that a specific integration exists; secondary action: contact Nymbus about an integration not listed.

## Who it's for
Primary: CTOs, product managers, and vendor evaluation teams at banks and credit unions checking integration coverage before committing to a demo or proposal. Secondary: current customers verifying roadmap status for upcoming integrations. Visitors arrive with a specific tool in mind — they want fast confirmation, not a sales pitch.

## What we're communicating
- Nymbus has a deep, pre-built partner ecosystem (73 integrations listed)
- Integrations span all critical banking functions: identity, payments, fraud, compliance, lending, servicing
- Coverage is split across SMB and Retail categories so visitors can filter to what's relevant
- Status indicators (Available / Roadmap) set honest expectations
- If a needed integration isn't listed, Nymbus is open to the conversation

## Tone & feel
Technical and utilitarian. This page is a reference tool, not a marketing page. The design should prioritize scanning and searching over reading. Consistent with "Built Without Limits" — the breadth of the ecosystem is the message.

## Must-haves / non-negotiables
- Filter tabs: All (73), SMB (21), Retail (52) — counts must stay current
- Live search/filter by partner name or function
- Partner cards: logo + function label + status badge (color-coded: green = Available, amber = Roadmap)
- Status badge color system must be consistent and accessible
- "Don't see your integration?" CTA at page bottom
- Partner logos must use only approved/confirmed assets — editorial note in current code warns against unconfirmed logos

## Interactive / visual elements
- Partner cards: colored top border indicates status (green = available, amber = roadmap)
- Search: real-time filter as user types — no submit required
- Category pills: instant filter; selected state clearly distinguished

## What to avoid
- Don't bury the search; it's the primary interaction pattern for this page
- Avoid making every card the same visual weight — status badge differentiation matters
- No long introductory copy before the grid; the hero is intentionally minimal
- Em-dashes
